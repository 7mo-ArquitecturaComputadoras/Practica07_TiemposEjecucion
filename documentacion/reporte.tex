\section{Introducción}

Esta práctica consiste en implementar el cálculo de la \textbf{norma euclídea} (también llamada módulo o longitud) de un vector de números reales, utilizando \textbf{lenguaje ensamblador x86} con la \textbf{Unidad de Punto Flotante (FPU)}. La función ensamblador es invocada desde un programa principal escrito en \textbf{C++}, que se encarga de solicitar los datos al usuario y presentar el resultado con seis decimales de precisión.

El objetivo va más allá de reproducir la operación aritmética: se busca comprender cómo la FPU acumula sumas de cuadrados en su pila interna, cómo se ejecuta una raíz cuadrada a nivel de hardware mediante la instrucción \texttt{FSQRT}, y cómo el procesador recorre un arreglo contiguo en memoria avanzando un puntero registro a registro.

\subsection*{¿Qué es la norma de un vector?}

Dado un vector $\vec{v} = (v_1, v_2, \ldots, v_N)$ de dimensión $N$, su \textbf{norma euclídea} (norma-2 o módulo) se define como:

\begin{equation}
    \|\vec{v}\| = \sqrt{\sum_{i=1}^{N} v_i^2} = \sqrt{v_1^2 + v_2^2 + \cdots + v_N^2}
\end{equation}

Geométricamente, la norma representa la \textbf{longitud del vector} en el espacio $n$-dimensional. Por ejemplo, para un vector de dimensión 3:

\begin{equation}
    \|(3,\; 4,\; 0)\| = \sqrt{3^2 + 4^2 + 0^2} = \sqrt{9 + 16} = \sqrt{25} = 5
\end{equation}

La norma siempre produce un \textbf{escalar no negativo}, y es igual a cero únicamente para el vector nulo.

\subsection*{¿Cómo se representan los vectores en memoria?}

En C++, un \texttt{vector<double>} almacena sus elementos de forma \textbf{contigua} en memoria. Cada elemento de tipo \texttt{double} ocupa \textbf{8 bytes} (64 bits, formato IEEE 754 de doble precisión). Si el primer elemento está en la dirección base $D$, los elementos sucesivos se encuentran en:

\begin{table}[H]
    \centering
    \begin{tabular}{@{}ccc@{}}
        \toprule
        \textbf{Elemento} & \textbf{Dirección} & \textbf{Desplazamiento} \\
        \midrule
        \texttt{vec[0]} & $D$       & $+0$ bytes \\
        \texttt{vec[1]} & $D + 8$   & $+8$ bytes \\
        \texttt{vec[2]} & $D + 16$  & $+16$ bytes \\
        \texttt{vec[i]} & $D + 8i$  & $+8i$ bytes \\
        \bottomrule
    \end{tabular}
    \caption{Distribución en memoria de un arreglo de \texttt{double}}
\end{table}

El ensamblador recorre estos elementos avanzando el puntero \texttt{ESI} en 8 bytes por iteración con la instrucción \texttt{ADD ESI, 8}.

\section{Código fuente}

\subsection{Programa principal — \texttt{main.cpp}}

\begin{lstlisting}[language=CPP, caption={main.cpp -- Programa principal en C++}]
// ============================================================
// Autor: Edson Joel Carrera Avila
// main.cpp
// ============================================================

#include <iostream>
#include <iomanip>
#include <vector>

using namespace std;

extern "C" double vectorNormal(double* vec, int N);

int main() {
    int n;

    cout << "Ingresa la dimension del vector (N): ";
    cin >> n;

    vector<double> vec(n);

    cout << "\n--- DATOS DEL VECTOR ---" << endl;
    for (int i = 0; i < n; i++) {
        cout << "Ingresa el valor para vec[" << i << "]: ";
        cin >> vec[i];
    }

    double norma = vectorNormal(vec.data(), n);

    cout << "\n----------------------------------------\n";
    cout << "Resultado de la Norma del Vector: "
         << fixed << setprecision(6) << norma << endl;

    return 0;
}
\end{lstlisting}

\subsection{Módulo ensamblador — \texttt{vectorNormal.asm}}

\begin{lstlisting}[language=MASM, caption={vectorNormal.asm -- Función ensamblador x86}]
; ============================================================
; Autor: Edson Joel Carrera Avila
; vectorNormal.asm
; ============================================================

.586
.model flat, c

; ============================================================
; SECCIÓN DE CÓDIGO (.code)
; ============================================================
.code
; --- Inicio del programa ---

; Parámetros: [EBP+8]  = puntero a vec (double*)
;             [EBP+12] = N (int)
vectorNormal PROC
    PUSH    EBP                     ; Guardamos el marco de pila actual
    MOV     EBP, ESP                ; Establecemos el nuevo marco de pila
    PUSH    ESI                     ; Preservamos ESI según la convención cdecl
    MOV     ESI, [EBP+8]            ; ESI apunta al inicio de vec
    MOV     ECX, [EBP+12]           ; ECX almacena la dimensión N (contador del ciclo)
    FLDZ                            ; Cargamos 0.0 en la pila FPU. ST(0) = acumulador
    TEST    ECX, ECX                ; Comparamos N con 0
    JLE     fin                     ; Si N <= 0, saltamos al final (retorna 0.0)

; --- Bucle de suma de cuadrados ---
bucle:
    FLD     QWORD PTR [ESI]         ; Cargamos vec[i] en ST(0). El acumulador baja a ST(1)
    FMUL    ST(0), ST(0)            ; Multiplicamos ST(0) por sí mismo. ST(0) = vec[i]^2
    FADDP   ST(1), ST(0)            ; Sumamos ST(0) al acumulador en ST(1) y desapilamos
    ADD     ESI, 8                  ; Avanzamos 8 bytes en vec (tamaño de un double)
    DEC     ECX                     ; Disminuimos el contador N
    JNZ     bucle                   ; Si ECX no es 0, repetimos el ciclo

; --- Raíz cuadrada ---
    FSQRT                           ; ST(0) = sqrt(suma_cuadrados)

; --- Fin del programa ---
fin:
    POP     ESI
    MOV     ESP, EBP
    POP     EBP
    RET
vectorNormal ENDP

END
\end{lstlisting}

\section{Instrucciones x86 utilizadas}

\begin{table}[H]
    \centering
    \begin{tabular}{@{}lp{9.5cm}@{}}
        \toprule
        \textbf{Instrucción} & \textbf{Qué hace} \\
        \midrule
        \texttt{PUSH / POP} & Coloca o retira un valor de 32 bits de la pila del sistema; se usan en el prólogo/epílogo y para preservar \texttt{ESI} según la convención \textit{cdecl}. \\
        \texttt{MOV}    & Copia un valor entre registros o entre registro y memoria. \\
        \texttt{ADD}    & Suma un valor inmediato a un registro; aquí avanza el puntero \texttt{ESI} en 8 bytes por iteración para acceder al siguiente \texttt{double}. \\
        \texttt{DEC}    & Decrementa el operando en 1; actualiza las banderas del procesador, lo que permite a \texttt{JNZ} detectar el fin del bucle. \\
        \texttt{TEST}   & Realiza una AND lógica sin guardar el resultado; solo actualiza las banderas para verificar si \texttt{ECX} es cero antes del primer ciclo. \\
        \texttt{JLE}    & Salta si el resultado indica ``menor o igual'' (\textit{jump if less or equal}); protege el bucle cuando $N \leq 0$. \\
        \texttt{JNZ}    & Salta si el resultado no es cero (\textit{jump if not zero}); mantiene el bucle mientras el contador \texttt{ECX} $\neq 0$. \\
        \texttt{RET}    & Retorna al llamador (C++); el resultado \texttt{double} permanece en \texttt{ST(0)} de la FPU. \\
        \texttt{FLDZ}   & Carga la constante 0.0 al tope de la pila FPU (\texttt{ST(0)}); inicializa el acumulador de la suma de cuadrados. \\
        \texttt{FLD}    & Carga (\textit{push}) un valor \texttt{QWORD} (64 bits / \texttt{double}) desde memoria al tope de la pila FPU; desplaza \texttt{ST(0)} a \texttt{ST(1)}. \\
        \texttt{FMUL}   & Multiplica \texttt{ST(0)} por el operando indicado; en este caso \texttt{ST(0),ST(0)} eleva al cuadrado el valor recién cargado. \\
        \texttt{FADDP}  & Suma \texttt{ST(0)} a \texttt{ST(1)} y hace \textit{pop}; el acumulador queda actualizado en lo que era \texttt{ST(1)}, que pasa a ser el nuevo \texttt{ST(0)}. \\
        \texttt{FSQRT}  & Calcula la raíz cuadrada del valor en \texttt{ST(0)} de forma \textbf{nativa en hardware} y almacena el resultado en el mismo registro; no requiere iteraciones por software. \\
        \bottomrule
    \end{tabular}
    \caption{Instrucciones x86 y FPU utilizadas en \texttt{vectorNormal}}
\end{table}

\section{El rol de los registros}

\subsection{Marco de pila y parámetros}

Al momento en que C++ invoca \texttt{vectorNormal}, coloca los dos argumentos en la pila del sistema en orden inverso (de derecha a izquierda, convención \textit{cdecl}). Después del prólogo \texttt{PUSH EBP / MOV EBP, ESP}, el mapa de la pila queda:

\begin{table}[H]
    \centering
    \begin{tabular}{@{}ccc@{}}
        \toprule
        \textbf{Dirección} & \textbf{Contenido} & \textbf{Tipo} \\
        \midrule
        \texttt{[EBP+12]} & \texttt{N}     & \texttt{int} (4 bytes) \\
        \texttt{[EBP+8]}  & \texttt{vec}   & \texttt{double*} (4 bytes en x86) \\
        \texttt{[EBP+4]}  & dirección de retorno & — \\
        \texttt{[EBP+0]}  & EBP anterior   & — \\
        \bottomrule
    \end{tabular}
    \caption{Mapa del marco de pila al inicio de \texttt{vectorNormal}}
\end{table}

A diferencia de la práctica anterior (producto punto con dos vectores), aquí solo se recibe \textbf{un puntero}, lo que simplifica el prólogo: únicamente es necesario preservar \texttt{ESI}, ya que \texttt{EDI} no se utiliza.

\subsection{Registros del procesador}

\begin{table}[H]
    \centering
    \begin{tabular}{@{}llp{7.5cm}@{}}
        \toprule
        \textbf{Registro} & \textbf{Rol} & \textbf{Descripción} \\
        \midrule
        \texttt{ESI} & Puntero al vector  & Apunta al elemento actual de \texttt{vec}; avanza 8 bytes por iteración para recorrer el arreglo contiguo en memoria. \\
        \texttt{ECX} & Contador de ciclo  & Almacena $N$ e inicia la cuenta regresiva hasta 0; al llegar a cero, \texttt{JNZ} no salta y el bucle termina. \\
        \texttt{ST(0)} & Tope de la pila FPU & Valor temporal: \texttt{vec[i]} recién cargado o su cuadrado; al terminar el bucle contiene la suma de cuadrados y después de \texttt{FSQRT} contiene la norma. \\
        \texttt{ST(1)} & Segundo registro FPU & Acumulador: suma parcial de $v_i^2$; solo existe mientras hay un elemento cargado en \texttt{ST(0)}. \\
        \bottomrule
    \end{tabular}
    \caption{Registros del procesador utilizados en \texttt{vectorNormal}}
\end{table}

\subsection{Estado de la pila FPU por instrucción}

La pila FPU tiene profundidad máxima de 8 registros (\texttt{ST(0)}--\texttt{ST(7)}). En este algoritmo se utiliza una profundidad máxima de 2. La siguiente tabla muestra cómo evoluciona en las instrucciones clave del bucle:

\begin{table}[H]
    \centering
    \begin{tabular}{@{}llll@{}}
        \toprule
        \textbf{Instrucción} & \textbf{ST(0)} & \textbf{ST(1)} & \textbf{Descripción} \\
        \midrule
        \texttt{FLDZ}              & 0.0         & vacío       & Inicializa el acumulador. \\
        \texttt{FLD [ESI]}         & $v_i$       & acum        & Carga el elemento; el acumulador baja a \texttt{ST(1)}. \\
        \texttt{FMUL ST(0),ST(0)}  & $v_i^2$     & acum        & Eleva al cuadrado sin mover la pila. \\
        \texttt{FADDP ST(1),ST(0)} & acum$+v_i^2$  & vacío     & Suma y hace \textit{pop}; la pila vuelve a profundidad 1. \\
        \texttt{FSQRT}             & $\sqrt{\text{acum}}$ & vacío & Raíz cuadrada en hardware; profundidad 1. \\
        \bottomrule
    \end{tabular}
    \caption{Estado de la pila FPU durante la ejecución de \texttt{vectorNormal}}
\end{table}

\section{Conclusiones}

\begin{itemize}
    \item La norma de un vector ilustra cómo el procesador puede ejecutar algoritmos matemáticos completos —incluyendo potenciación, acumulación y raíz cuadrada— \textbf{íntegramente mediante la FPU}, sin depender de bibliotecas de punto flotante de software. La instrucción \texttt{FSQRT} delega la operación directamente a hardware dedicado, lo que la hace eficiente y precisa.

    \item El patrón \textbf{\texttt{FLD} → \texttt{FMUL ST(0),ST(0)} → \texttt{FADDP ST(1),ST(0)}} mantiene la pila FPU con profundidad máxima de 2 durante todo el bucle: un elemento para el valor al cuadrado y otro para el acumulador. Esta gestión cuidadosa evita desbordamientos de la pila de 8 niveles y resulta más eficiente que cargar y descargar el acumulador en cada iteración.

    \item La diferencia respecto al \textbf{producto punto} (Práctica 05) pone de manifiesto la flexibilidad de la FPU: donde antes se necesitaban dos punteros (\texttt{ESI} y \texttt{EDI}) y una multiplicación cruzada entre vectores, aquí basta con \textbf{un solo puntero} y una auto-multiplicación (\texttt{FMUL ST(0),ST(0)}), demostrando que pequeñas variaciones en el algoritmo tienen un impacto directo en el número de registros utilizados.

    \item La \textbf{verificación de $N \leq 0$} con \texttt{TEST} y \texttt{JLE} es indispensable: si se omitiera y se pasara un valor negativo, \texttt{DEC} / \texttt{JNZ} ejecutarían el bucle aproximadamente $2^{32}$ veces, leyendo memoria fuera del arreglo y produciendo resultados indefinidos o corrupción de datos.

    \item Preservar únicamente \textbf{\texttt{ESI}} (y no \texttt{EDI}, que no se usa) muestra el principio de \textbf{mínima perturbación}: la convención \textit{cdecl} obliga a guardar los registros que se modifiquen, pero no los que permanecen intactos, reduciendo el número de operaciones de pila innecesarias.
\end{itemize}
